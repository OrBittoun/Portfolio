Wayzy

My team and I developed a smart algorithm for real-time multi-order delivery optimization during a 2-day hackathon, outperforming common market solutions.
The algorithm scores orders based on urgency and proximity using live data from the Google Maps API.
Currently, we're expanding it into a full-scale product.

Wayzy

Each order is assigned a final score based on the weighting of the two components (45% proximity, 55% urgency), with the aim of separating urgent but distant orders from close and less urgent orders.

The system selects the most urgent order (called the "anchor") from the latest restaurants, and examines which additional orders can be optimally combined in the route, to the maximum number set by the user.

The selected route undergoes final optimization, which arranges the order of deliveries so that the travel time is minimal.

The project was built entirely in Python, using libraries such as pandas for data processing.

This system provides an independent, smart and flexible tool for couriers, with or without a connection, among other things, and gives a real competitive advantage in planning accurate and efficient delivery routes.
